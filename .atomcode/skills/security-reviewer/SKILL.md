# 安全审查 Skill

对代码进行安全审查，确保不泄露凭证、防止注入攻击、保护用户数据。

## 触发条件

- 手动调用：`/security-review <file-or-directory-path>`
- 编辑涉及以下内容的文件时自动提示：
  - API 路由（`app/api/**/route.ts`）
  - 数据库操作
  - 用户输入处理
  - 外部 API 调用

## 检查清单

1. **凭证泄露**：检查是否有 API key、密码、token 硬编码
2. **注入攻击**：SQL 注入、NoSQL 注入、命令注入
3. **XSS 防护**：用户输入是否经过转义处理
4. **CSRF 防护**：写操作是否有 CSRF token 验证
5. **权限检查**：敏感操作是否有用户身份验证
6. **数据验证**：输入数据是否有 schema 校验
7. **敏感信息**：日志、错误信息不暴露内部细节
